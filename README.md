# 🚀 QA Learning Path – Kanban de Estudos

> Roadmap prático e estruturado de estudos em **Quality Assurance (QA)**, utilizando **GitHub Projects (Kanban)** para organização, rastreabilidade e evolução contínua.

---

## 🧭 Visão Geral

Este repositório foi criado para **organizar, documentar e acompanhar minha evolução técnica como QA**, com foco em práticas reais do mercado.

O Kanban aqui estruturado cobre tanto **fundamentos teóricos** quanto **aplicação prática**, abrangendo:

- ✅ Testes Manuais  
- ✅ Testes de API  
- ✅ Automação de Testes  
- ✅ Qualidade Não Funcional  
- ✅ Dados, Mobile e Sistemas Legados  

Além do aprendizado, este repositório também funciona como **material de portfólio profissional**.

---

## 🎯 Objetivos do Kanban

- Centralizar estudos técnicos em QA  
- Transformar teoria em prática real  
- Criar histórico de aprendizado rastreável  
- Evidenciar disciplina, planejamento e evolução contínua  
- Facilitar revisões e reaproveitamento de conhecimento  

---

## 🧩 Estrutura do Kanban (GitHub Projects)

O fluxo do Kanban segue as etapas abaixo:

| Coluna | Descrição |
|------|----------|
| 📦 **Backlog de Estudos** | Tópicos mapeados e ainda não iniciados |
| 🗂️ **Planejado** | Estudos priorizados |
| 📖 **Em Estudo** | Fase teórica em andamento |
| 🛠️ **Em Prática** | Aplicação prática / hands-on |
| ✅ **Concluído** | Estudo finalizado |
| 🔁 **Revisar / Reforçar** | Conteúdos para revisão |

---

## 🚩 Milestones (Macroetapas)

Os **Milestones** representam grandes blocos de aprendizado:

1. Fundamentos de Testes de Software  
2. Testes Manuais Avançados  
3. BDD e Gherkin  
4. Testes de API  
5. Automação de Testes  
6. Qualidade Não Funcional  
7. Dados, Mobile e Mainframe  

Cada milestone contém várias **Issues (tarefas de estudo)**.

---

## 🧪 Issues (Tarefas de Estudo)

Cada **Issue** representa um tópico específico de aprendizado.

As issues são compostas por:
- Título claro e objetivo  
- Labels (tags) técnicas  
- Checklist interno para acompanhamento
